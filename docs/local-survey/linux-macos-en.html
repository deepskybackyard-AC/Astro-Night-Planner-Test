Linux/macOS: native Local Survey Server 1.1.3

This variant is the additional solution for Linux and macOS. It does not replace the Windows application and does not require Wine or Python.

What the program does

  • It serves one shared survey root folder through a local HTTP server.
  • Default survey URL: http://127.0.0.1:8765/
  • Default configuration interface: http://127.0.0.1:8776/
  • The interface runs in the browser and behaves the same on Linux and macOS.

Start on Linux

  1. Extract the package.
  2. Make the matching binary executable.
  3. Start it with the survey root folder.
chmod +x bin/anp-local-survey-server-linux-amd64
./bin/anp-local-survey-server-linux-amd64 --root "$HOME/AstroSurveys" --port 8765 --open

Start on macOS

  1. Extract the package.
  2. Make the matching binary executable.
  3. On first launch macOS may show a security warning. Allow the launch in Finder or in Privacy & Security.
  4. Start it with the survey root folder.
chmod +x bin/anp-local-survey-server-darwin-arm64
./bin/anp-local-survey-server-darwin-arm64 --root "$HOME/AstroSurveys" --port 8765 --open

Autostart and stopping

Linux: see the systemd user-service guide. macOS: see the LaunchAgent guide. In a terminal the server can be stopped with Ctrl+C. The browser interface also provides Exit program.

Folder selection and survey downloads

The root folder and download target can be selected with an operating-system folder dialog. All surveys built into the app are listed with default source URLs and can be downloaded, paused, resumed, cancelled and validated in the browser interface.

Downloading another HiPS survey

Select Other / custom HiPS survey in the download manager. Enter the complete HiPS source URL and a relative target path inside the survey root, for example custom/my-survey/.

A complete recursive download is possible only when the source server provides a directory listing and the download is technically and legally permitted. Very large surveys may require substantial storage and download time.

Windows configuration from version 1.0

On first start, server 1.1.3 searches for the previous ANP-Local-Survey-Server-config.json in the program folder, current folder, user folder and AppData. If found, survey root, port, language and autostart settings are migrated to the new configuration. The old file is retained as a backup.
